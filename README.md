# Khuroson MES (Manufacturing Execution System)

Индустриальная MES-система управления производством строительных материалов (на примере цеха по выпуску шлакоблоков). Проект реализует строгий учет движения сырья на складе, управление рецептурами готовых изделий и автоматическое списание компонентов в момент фиксации выпуска продукции.

## 🚀 Ключевые особенности и рефакторинг

В ходе разработки проект был успешно переведен с прототипа на промышленную архитектуру:
- **Переход на PostgreSQL**: База данных переведена с SQLite на PostgreSQL для поддержки конкурентных асинхронных транзакций и полноценной работы с миграциями.
- **Сервисный слой (Service Layer)**: Бизнес-логика полностью изолирована от API-контроллеров в выделенные сервисы `warehouse.py` (склад) и `manufacturing.py` (производство).
- **Защита от дубликатов**: На уровне СУБД внедрен составной уникальный индекс (`UniqueConstraint`), исключающий появление дублирующихся рецептов и повторных списаний сырья.
- **Оптимизация SQL (N+1 Fix)**: Расчет остатков на складе переписан с циклического выполнения запросов на единый эффективный SQL-запрос с агрегацией (`func.sum`) и группировкой.
- **Автоматические тесты**: Написан сквозной интеграционный авто-тест (`pytest` + `httpx`), проверяющий весь жизненный цикл системы: от оприходования сырья до контроля блокировки дублей и точного финального баланса.

## 📂 Архитектура проекта

Проект организован по модульному принципу с четким разделением зон ответственности:

```text
khuroson_mes/
├── .env.example          # Шаблон конфигурации окружения
├── .gitignore            # Исключения для Git (секреты, venv)
├── README.md             # Инструкция и документация проекта
├── requirements.txt      # Зависимости (FastAPI, SQLAlchemy 2.0, asyncpg)
├── alembic/              # Миграции базы данных (PostgreSQL)
└── src/                  # Исходный код бэкенда
    ├── main.py           # Точка входа в приложение FastAPI
    ├── core/             # Системное ядро (база данных, чтение .env через Pydantic)
    ├── models/           # SQLAlchemy-модели (справочники, транзакции, логи)
    ├── schemas/          # Схемы валидации Pydantic v2 (типы данных, контракты API)
    ├── services/         # Бизнес-логика (warehouse.py, manufacturing.py)
    └── api/              # Маршрутизация API (эндпоинты для материалов и производства)
```

## 💻 Инструкция по локальному запуску

### 1. Клонирование репозитория и настройка окружения
```bash
git clone https://github.com
cd khuroson-mes

# Создание и активация виртуального окружения
python3 -m venv venv
source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка переменных окружения
Создайте файл `.env` в корне проекта на основе шаблона `.env.example`:
```ini
PROJECT_NAME="Khuroson MES"
DATABASE_URL="postgresql+asyncpg://mes_user:your_password@localhost:5432/khuroson_mes"
```

### 3. Запуск миграций базы данных
Убедитесь, что ваш сервер PostgreSQL запущен и база данных создана, затем примените структуру таблиц:
```bash
alembic upgrade head
```

### 4. Запуск сервера разработки
```bash
uvicorn src.main:app --reload
```
После запуска интерактивная документация Swagger API будет доступна по адресу: `http://127.0.0.1`

## 🧪 Запуск автоматических тестов

Интеграционные тесты проверяют корректность проведения транзакций, контроль остатков сырья и работу валидации дубликатов рецептур.

При запущенном Uvicorn сервере выполните команду:
```bash
pytest test_production_api.py -v -s
```
